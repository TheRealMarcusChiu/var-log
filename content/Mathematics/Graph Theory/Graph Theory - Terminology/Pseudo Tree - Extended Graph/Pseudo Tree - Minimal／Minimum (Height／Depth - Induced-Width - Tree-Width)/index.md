the [[Problems|problem]] of <font style="color: rgb(128,128,128);"><em>finding a [[Pseudo Tree - Extended Graph|pseudo tree]] of (minimal height/depth </em>or<em> minimal induced width)</em></font> is known to be [[Complexity Classes|NP-complete]]. This problem is found in various tasks (e.g. finding the smallest possible [[AND／OR Search Graphs (AOG)|AND/OR Search Graph]]). Greedy algorithms and polynomial-time heuristic schemes are available

# 2 Schemes

Given a [[Graphical Models|graphical model]] 𝒢 and an ordering 𝑑:

- the [[DFS Spanning Tree (DST)|DFS Spanning Tree]] generated by traversing the [[(Ordered Graph - Width) - (Induced Graph - Induced Width) - (Conditional Induced Graph - Conditional Induced Width)|induced-order graph]] starting at the first variable of its ordering 𝑑 is a pseudo tree
- the [[Hypergraphs - Primal Graphs - Dual Graphs - Cluster Graphs - Cluster Trees - Factor Graphs|hypergraph tree-decomposition]] / [[Probabilistic Inference - Bucket Tree Elimination (BTE) - Collect & Distribute Algorithm|bucket tree]] derived from the [[(Ordered Graph - Width) - (Induced Graph - Induced Width) - (Conditional Induced Graph - Conditional Induced Width)|induced ordered graph]] along 𝑑 of 𝒢, 𝑇=(𝑋, 𝐸) with 𝐸 = {(𝑋<sub>𝑖</sub>,𝑋<sub>𝑗</sub>) | (𝐵<sub>𝑋<sub>𝑖</sub></sub>,𝐵<sub>𝑋<sub>𝑗</sub></sub>) ∊ 𝑏𝑢𝑐𝑘𝑒𝑡 - 𝑡𝑟𝑒𝑒} is a pseudo tree of 𝒢

# Greedy Search

eliminate node with the smallest cost

- <strong>min-neighbors/degree heuristic</strong> - number of neighbors in the current graph
- <strong>min-weight heuristic</strong> - weight (number of values) of factor formed
- <strong>min-fill heuristic</strong> - number of new fill edges to the current graph
- <strong>weighted min-fill heuristic</strong> - total weight of new fill edges (edge weight = product of weights of the 2 nodes)

Since an induced-ordered graph can be a starting point in generating a pseudo tree, the question is if the min-fill ordering heuristic which appears to be quite good for finding small induced-width is also good for finding pseudo trees with small heights

It is generally observed that:

- <strong>min-fill heuristic</strong> generates lower [[(Ordered Graph - Width) - (Induced Graph - Induced Width) - (Conditional Induced Graph - Conditional Induced Width)|induced width]] pseudo trees
- <strong>hypergraph heuristic</strong> produces much smaller [[Tree - Depth & Height|height/depth]] pseudo trees
