Jenkins 2 comes with a great feature called <strong><em>Pipelines</em></strong>, which is very extensible when we need to define a [[Continuous Integration (CI) - Continuous Delivery (CD) - Continuous Deployment|continuous integration/delivery/deployment]] environment for a project.

A Pipeline is another way of defining some Jenkins steps using code, and automate the process of deploying software.

It's using a [Domain Specific Language(DSL)](https://jenkins.io/doc/book/pipeline/syntax/) with two different syntaxes:

- Declarative Pipeline
- Scripted Pipeline
