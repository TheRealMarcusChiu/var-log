Maven can be invoked with:
1. a phase (e.g <code>clean</code>, <code>package</code>)
2. [<code>\<plugin-prefix\>:\<goal\></code>](https://maven.apache.org/guides/introduction/introduction-to-plugin-prefix-mapping.html) (e.g. <code>dependency:copy-dependencies</code>)
3. <code>\<plugin-group-id\>:\<plugin-artifact-id\>\[:\<plugin-version\>\]:\<goal\></code> (e.g. <code>org.apache.maven.plugins:maven-compiler-plugin:3.7.0:compile</code>)

with one or more combinations of any or all:
- <code>mvn clean dependency:copy-dependencies package</code>

# More Details
> [!tabs]
>
> === Lifecycle
>
> maven contains 3 built-in life cycles:
> - <strong>default</strong> <strong>(build)</strong>: the main life cycle as it's responsible for project deployment
> - <strong>clean</strong>: to clean the project and remove all files generated by the previous build
> - <strong>site</strong>: to create the project's site documentation
>
> Each life cycle consists of a sequence of <strong>phases</strong>:
> - <strong>default</strong> build life cycle consists of 23 phases as it's the main build lifecycle
> - <strong>clean</strong> life cycle consists of 3 phases
> - <strong>site</strong> lifecycle is made up of 4 phases
>
> === Phase
>
> A <strong>phase</strong> represents a stage in the Maven build lifecycle
>
> Here are some of the most important phases in the <strong>default</strong> build lifecycle:
> - <strong>validate</strong>: check if all information necessary for the build is available
> - <strong>compile</strong>: compile the <em>source-code</em>
> - <strong>test-compile</strong>: compile the <em>test-source-code</em>
> - <strong>test</strong>: run unit tests
> - <strong>package</strong>: package compiled <em>source-code</em> into the distributable format (jar, war, …)
> - <strong>integration-test</strong>: process and deploy the package if needed to run integration tests
> - <strong>install</strong>: install the package to a local repository
> - <strong>deploy:</strong> copy the package to the remote repository
>
> For the full list of each lifecycle's phases, check out the [Maven Reference](https://maven.apache.org/guides/introduction/introduction-to-the-lifecycle.html#Lifecycle_Reference).
>
> commands:
> - Run lifecycle up to phase: <code><font style="color: rgb(128,128,128);">mvn \$PHASE</font></code>
>
> === Goal
>
> Each phase is a sequence of <strong>goals</strong>
>
> Here are some of the phases and default goals bound to them:
> - <strong><em>compiler:compile</em></strong> – the <em>compile</em> goal from the <em>compiler</em> plugin is bound to the <em>compile</em> phase
> - <strong><em>compiler:testCompile</em></strong> is bound to the <em>test-compile</em> phase
> - <strong><em>surefire:test</em></strong> is bound to <em>test</em> phase
> - <strong><em>install:install</em></strong> is bound to <em>install</em> phase
> - <strong><em>jar:jar</em></strong> and <strong><em>war:war</em></strong> is bound to <em>package</em> phase
>
> commands:
> - List all goals bound to a phase and their plugins: <code><font style="color: rgb(128,128,128);">mvn help:describe -Dcmd=\$PHASE</font></code>
>
> === Plugin
>
> A <strong>plugin</strong> is a group of goals. However, these goals aren't necessarily all bound to the same phase
>
> commands:
> - list all goals in a specific plugin: <code><font style="color: rgb(128,128,128);">mvn \$PLUGIN:help</font></code>
> - run a specific goal, without executing its entire phase (and the preceding phases): <code><font style="color: rgb(128,128,128);">mvn \$PLUGIN:\$GOAL</font></code>
>
> <font style="color: rgb(83,90,96);">A plugin is a collection of </font><strong>goals</strong><font style="color: rgb(83,90,96);"> also called MOJO (</font><strong>M</strong><font style="color: rgb(83,90,96);">aven </font><strong>O</strong><font style="color: rgb(83,90,96);">ld </font><strong>J</strong><font style="color: rgb(83,90,96);">ava </font><strong>O</strong><font style="color: rgb(83,90,96);">bject).</font>
>
> <font style="color: rgb(83,90,96);">Analogy: Plugin is a class and goals are methods within the class</font>
