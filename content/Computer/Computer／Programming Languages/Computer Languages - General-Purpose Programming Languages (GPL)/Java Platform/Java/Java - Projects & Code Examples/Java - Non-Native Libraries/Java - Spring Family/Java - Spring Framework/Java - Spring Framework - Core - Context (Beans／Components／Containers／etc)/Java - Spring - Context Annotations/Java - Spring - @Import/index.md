###### @Import

- takes in both [[Java - Spring - @Configuration|@Configuration]] classes and [[Java - Spring - @Bean @Component|@Component]] classes
- Spring generally promotes the [convention-over-configuration](https://en.wikipedia.org/wiki/Convention_over_configuration) approach. Making an analogy with our annotations, @<em>ComponentScan</em> is more like convention, while @<em>Import</em> looks like configuration
