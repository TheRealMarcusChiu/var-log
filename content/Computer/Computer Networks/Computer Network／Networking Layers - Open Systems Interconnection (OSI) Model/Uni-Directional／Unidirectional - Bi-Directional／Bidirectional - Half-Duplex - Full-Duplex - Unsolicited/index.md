```excerpt
- <strong>uni-directional </strong>- data can only be sent in 1 direction
- <strong>bi-directional</strong> - data can be sent in both directions
	- <strong>half-duplex</strong> - data can be sent in both directions, but not at the same time
	- <strong>full-duplex</strong> - data can be sent in both directions at the same time
- <strong>unsolicited</strong> - data can be sent to the receiver, without the receiver's previous request ([[WebSockets]], [[Full Cone NAT - Port Forwarding|port forwarding]], etc)
```

^excerpt
